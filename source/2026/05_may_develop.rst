5月発展：買い物の計算グラフからニューラルネットワークへ
========================================================

テーマ
------

買い物の計算を使って、計算グラフと誤差逆伝播法の考え方を知る

この回のねらい
--------------

5月本編では、入力・計算・条件分岐を使ってレジアプリを作りました。

この発展編では、その計算を **「計算グラフ」** として見直し、
ニューラルネットワークにつながる考え方を体験します。

ここで練習したいのは、次の3つです。

- 計算を部品に分けて考える
- 出力から逆向きに影響をたどる
- その考え方がニューラルネットワークの学習につながると知る


内容
----

1. なぜ買い物の例で学ぶのか
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

ニューラルネットワークでは、
たくさんの計算を重ねて最後に予測結果を出します。

ただ、最初から複雑なモデルを見ると、
「どこで何を計算しているのか」
「なぜ逆向きにたどる必要があるのか」
が見えにくくなります。

そこで、まずは身近な買い物の計算を使います。

- 個数 × 値段
- 商品ごとの金額を足す
- 最後に消費税をかける

このように、掛け算と足し算が順番につながっているので、
計算グラフの形がとても見やすくなります。


2. 計算グラフとは
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**計算グラフ** とは、計算の流れを
「ノード」と「矢印」で表したものです。

たとえば、りんごとみかんを買ったとします。

.. code-block:: text

   apple_total = apple_price * apple_count
   orange_total = orange_price * orange_count
   subtotal = apple_total + orange_total
   total = subtotal * tax

これを流れで見ると、次のようになります。

.. code-block:: text

   apple_price --\
                  × --> apple_total --\
   apple_count --/                     \
                                        + --> subtotal --> × --> total
   orange_price --\                     /                  ^
                   × --> orange_total --/                   |
   orange_count --/                                        tax

このように、**1つ前の計算結果が次の計算に渡されていく**
という形で表せます。


3. まずは順方向に計算する
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

次の値を使ってみましょう。

- りんご: 100円を2個
- みかん: 80円を3個
- 消費税: 1.1

順番に計算すると、次のようになります。

.. code-block:: text

   apple_total = 100 * 2 = 200
   orange_total = 80 * 3 = 240
   subtotal = 200 + 240 = 440
   total = 440 * 1.1 = 484

このように、入力から出力へ向かって計算する流れを
**順伝播（forward）** と呼びます。

ニューラルネットワークでも、まずはこの順伝播で
予測値を計算します。


4. 逆向きにたどると何が分かるか
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

ここで知りたいのは、
**「合計金額が少し変わるとしたら、どの値がどれだけ影響するか」**
です。

これを逆向きにたどる考え方が、
**誤差逆伝播法（backpropagation）** の土台になります。

まず、出力 ``total`` から「影響の大きさ」を 1 としてスタートします。

.. code-block:: text

   dtotal/dtotal = 1

最後の式は ``total = subtotal * tax`` です。
掛け算では、片方へ戻すときにもう片方の値を掛けます。

.. code-block:: text

   dtotal/dsubtotal = 1 * tax = 1.1
   dtotal/dtax = 1 * subtotal = 440

つまり、

- ``subtotal`` が 1 増えると、``total`` は 1.1 増える
- ``tax`` が少し増えると、``total`` は 440 の大きさで変わる

と読めます。

次に、``subtotal = apple_total + orange_total`` です。
足し算では、影響はそのまま分かれます。

.. code-block:: text

   dtotal/dapple_total = 1.1
   dtotal/dorange_total = 1.1

さらに、``apple_total = apple_price * apple_count`` に戻ると、

.. code-block:: text

   dtotal/dapple_price = 1.1 * apple_count = 2.2
   dtotal/dapple_count = 1.1 * apple_price = 110

となります。

これは、

- りんご1個の値段が1円上がると、合計は2.2円増える
- りんごの個数が1増える向きに変わると、合計は110円分変わる

という意味です。

この **「後ろの影響を前へ配る」** 考え方が、
逆伝播の核心です。


5. 誤差逆伝播法は何をしているのか
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

ニューラルネットワークでは、出力そのものよりも
**正解とのずれ（誤差、損失）** を小さくしたいと考えます。

そのため、実際には

1. 順伝播で予測を出す
2. 損失関数で誤差を計算する
3. 逆伝播で各重みの勾配を求める
4. 勾配を使って重みを少し更新する

という流れで学習します。

ここでの「各重みに対して、誤差がどれだけ変わるか」を表すのが勾配です。
誤差逆伝播法は、その勾配を**計算グラフを逆向きにたどって効率よく求める方法**
だと考えるとつかみやすくなります。


6. 計算グラフと機械学習の関係
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

計算グラフそのものは、まだ機械学習ではありません。
計算のつながりを整理して見える形にしたものです。

機械学習になるのは、その計算グラフの中にある
**重み** をデータに合わせて調整し始めたときです。

たとえばニューラルネットワークでは、次のような計算が並びます。

.. code-block:: text

   入力 -> 重み付き和 -> 活性化関数 -> ... -> 出力 -> 損失

この全体も、大きな計算グラフです。
そして、損失から逆向きに勾配を流して、
各重みを少しずつ直していくのが学習です。


7. 最初に押さえたいポイント
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

ニューラルネットワークを学び始めるときは、
まず次の3点をつかめれば十分です。

- 順伝播は「入力から出力へ」計算する流れ
- 逆伝播は「出力の影響を前へ戻す」流れ
- 掛け算と足し算の逆向きルールを理解すると、全体が追いやすくなる

買い物の例でこの流れに慣れておくと、
その先で出てくる層、重み、活性化関数、損失関数も
「複雑な新しいもの」ではなく
**計算グラフの部品が増えたもの**
として見られるようになります。
